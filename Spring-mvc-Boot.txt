#What is Spring?
###################

What is Dependency Injection (DI) ?
#######################################################
-> Dependency Injection is a design pattern used by Spring Boot to automatically create and manage objects (dependencies) your classes need ‚Äî without you manually creating them.
-> Dependency Injection is a technique where objects (dependencies) are injected into a class by the framework (Spring) at runtime rather than the   object creating them itself.

In simple words:
Class will not create its own objects ‚Äî Spring will give those objects.  
Analogy:
------------
Imagine you're building a house. You don‚Äôt make the bricks yourself ‚Äî you ask someone to provide them.
Similarly, Spring Boot provides the objects your app needs.


Why Use DI?
###############
‚ÄúWe use DI to make the application loosely coupled, easier to test, easy to maintain, and to let Spring manage object creation instead of manual new() operations.‚Äù


What is Loose Coupling?
############################3
Loose coupling means classes are independent of each other.
ex -
class StudentService {
    private StudentRepository repo;

    @Autowired
    public StudentService(StudentRepository repo) { // dependency injected
        this.repo = repo;
    }
}
Now StudentService doesn‚Äôt care how repo is created.


What is Tight Coupling?
#########################
Tight coupling means one class is highly dependent on another class‚Äôs implementation.
The classes are strongly connected ‚Äî if one changes, the other must change too.
exex-
class StudentService {
    StudentRepository repo = new StudentRepository(); // tightly coupled   ,If Repository changes, you must modify StudentService.
}


What is Spring IOC(INVERSION OF CONTROL)?
##############################################3
-> Spring IOC Container is the core of the Spring Framework.
-> It manages the lifecycle and Creation of application objects (called beans).
-> It creates, configures, and wires objects automatically based on configuration
-> Helps implement Dependency Injection (DI) easily.
-> Promotes loose coupling because your classes don‚Äôt instantiate their dependencies directly.


Spring Bean Life Cycle
########################
Bean is Created---->Dependencies Injected (@Autowired)----->Execution(Bean is use by the application)------>Destruction of the bean
‚ÄúA Spring bean goes through four phases:
Spring creates the bean, injects dependencies using @Autowired, the bean is then used by the application, and finally Spring destroys it when the container shuts down. The entire lifecycle is automatically managed by the Spring IoC container.‚Äù



What is Bean Scope?'
#######################
Bean Scope refers to the lifecycle and visibility of a bean ‚Äî meaning how often a bean object is created and how long it will remain alive within the Spring container.
Common Bean Scopes in Spring:

1.Singleton(Default) - Only one instance per Spring container it is shared across all application

2.Prototype - Creates a new bean instance every time it is requested

3.Request (Web Only) -One bean instance per HTTP request

4.Session (Web Only) -One bean instance per HTTP session

5.Application (Web Only)- One bean instance per ServletContext


what is Bean factory

what is ApplicationContext?

ApplicationContext is the fully-featured Spring container that manages beans with advanced capabilities like AOP, event handling, and eager loading. It is preferred over BeanFactory in modern Spring applications

Why We Use Constructor-Based Dependency Injection?
###################################################
Constructor-based DI is considered the best practice in Spring because it provides stronger safety, clarity, and testability.
‚ÄúConstructor-based DI is preferred because it makes dependencies mandatory, promotes immutability, prevents null values, and makes unit testing easy.
Promotes Immutability means-  All dependencies are final and cannot be changed after object creation.
mandatory dependencies means -If Repo is missing, the class cannot be created, preventing runtime errors.


Why Don‚Äôt We Use @Autowired (Field Injection) in Modern Spring?
####################################################################33
We avoid @Autowired on fields because it makes the class hard to test, hides dependencies, prevents immutability, and tightly couples code to Spring.

What is Status Code
###################
Status codes in Spring Boot are HTTP response codes (like 200, 400, 404, 500) sent by the server to indicate success, client error, or server error in a request.

Status Code: 201 It  indicates that a resource has been successfully created as a result of a POST request.

Status Code: 200 ‚Äì OK --> means The request was successful, and the server has returned the expected response.

Status Code: 404 ‚Äì Not Found --> means The resource the client is trying to access does not exist on the server.

Status Code : 500 ‚Äì Internal Server Error  means --> The server encountered an unexpected error while processing the request.
Causes:
NullPointerException
Database connection error

Status Code : 403 ‚Äì Forbidden  --> means The server understood the request, the resource exists, but the client is not allowed to access it. because User does NOT have required permissions 
This is an authorization error, not an authentication error.

####################
