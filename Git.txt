üí¨ 1Ô∏è‚É£ ‚ÄúWhat is a merge conflict in Git?‚Äù

Simple answer:

A merge conflict happens when two branches change the same part of a file differently, and Git doesn‚Äôt know which version to keep.
We must manually review and fix it before completing the merge.



‚ÄúHave you faced a merge conflict? What did you do?‚Äù
--------------------------------------------------
Real-life style answer:

Yes, when two teammates and I updated the same config file, Git showed a merge conflict.
I checked both versions, discussed with the team, kept the correct settings, and committed the resolved version.
Then we made sure to pull the latest code before new changes to avoid it again.


üí¨ 3Ô∏è‚É£ ‚ÄúHow do you resolve a merge conflict?‚Äù

Answer:

Git marks the conflicting parts with <<<<<<<, =======, and >>>>>>> in the file.

I open the file, decide which code to keep (mine, theirs, or both),

Then I run:

git add <filename>
git commit


to mark it as resolved.
